# Implementation Plan
**Feature:** AI Coding Assistant — Git and GitHub Integration

## Overview
This plan outlines the systematic implementation of the Git and GitHub integration specification. It addresses the missing foundational abstraction layers and converts the naive `git-server` script into a robust, context-aware, safety-first workflow orchestrated by the `AgentRunner`.

## Phase 1: Local Git Safety Baseline & Core Services

### 1. `GitService` & Checkpoints
- **[NEW]** `src/git/GitService.ts`: Implement `GitService` interface from the spec (repository discovery, status ingestion, diff caching).
- **[NEW]** `src/git/CheckpointManager.ts`: Introduce shadow checkpoints.
  - *Implementation Detail:* We will initialize a separate, hidden `.bormagi/shadow.git` bare/worktree hybrid, or use hidden refs (`refs/bormagi/checkpoints/`) in the primary repo to store state before every edit sequence.

### 2. Validation & Repair Loop
- **[NEW]** `src/git/ValidationService.ts`: Executes configurable formatters, linters, and tests.
- **[NEW]** `src/git/CommitProposalGenerator.ts`: Generates structured, diff-grounded commit messages (Conventional Commits) automatically upon task completion.

### 3. Orchestration Refactoring
- **[MODIFY]** `src/agents/AgentRunner.ts`:
  - Fetch `RepoCapabilities` at session start.
  - Implement the "pre-edit pipeline" (FR-020): snapshot the repository state before file modifications.
  - Implement the "post-edit pipeline" (FR-021): run validation, assemble diff summaries, propose commit.

## Phase 2: Repository Instructions

### 1. Instruction Resolver
- **[NEW]** `src/context/InstructionResolver.ts`:
  - Locate and parse `AGENTS.md`, `.github/copilot-instructions.md`, and any files in `.github/instructions/**/*.instructions.md`.
  - Process YAML/Frontmatter or comment blocks to identify `applyTo` glob patterns for path-specific injection.
  - Merge the instructions with size caps (FR-052).

### 2. Context Injection
- **[MODIFY]** `src/context/PromptBuilder.ts` (or `AgentRunner.ts`):
  - Inject the merged text output from `InstructionResolver` into the `stablePrefix` or equivalent token budget slot.

## Phase 3: GitHub PR and Review Integration

### 1. GitHub Service
- **[NEW]** `src/github/GitHubService.ts`:
  - Replace raw `gh` CLI calls in `git-server.ts` with a robust service using Octokit (or safe CLI wrappers) and VS Code Authentication (`vscode.authentication`).
  - Implement PR draft creation, PR artifact generators (summary, testing checklist from diff), and check-run reading.

### 2. Review Loop Orchestration
- **[NEW]** `src/github/ReviewRemediationLoop.ts`: Parse PR review comments, map them to local context, and run targeted fixes bounded by a retry budget (FR-057).

## Phase 4: UI and Promotion UX

### 1. UI Extensions
- **[MODIFY]** `src/ui/chat/ChatView.ts` & `chat.html`:
  - Surface the dynamic branch name, dirty state, and "Last Checkpoint" info.
- **[NEW]** `src/ui/CheckpointPanel.ts`: Webview panel to visualize the shadow checkpoint history with a "Restore" button.

## Rollout & Backward Compatibility
- **Backward Compatibility:** Existing `git-server.ts` commands will remain temporarily for explicit user requests, but automated orchestration will shift to `GitService.ts` to enforce policy.
- **Safety:** Edits to dirty trees will automatically store a protective checkpoint before applying LLM edits.

## Automated Testing
- End-to-end integration tests for `CheckpointManager.ts`.
- Unit tests for `InstructionResolver.ts` path-matching rules.
- Mocked tests for `CommitProposalGenerator.ts`.
