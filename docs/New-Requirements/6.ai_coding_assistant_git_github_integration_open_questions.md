# Open Questions
**Feature:** AI Coding Assistant — Git and GitHub Integration

1. **Shadow Checkpoints Storage Mechanism:**
   The spec recommends "shadow Git mode" for checkpoints. Should we implement this by:
   - **Option A (Hidden Refs):** Using the main repository's `.git` folder but writing commits to isolated, hidden refs (e.g., `refs/bormagi/checkpoints/*`) so they stay out of the `git log` but reuse the object database?
   - **Option B (Separate Repo):** Initializing a completely separate Git repository inside `.bormagi/shadow.git` and syncing the working tree over? Option A is much faster and cleaner for large repos but risks user confusion if they poke at raw refs.

2. **Validation Fallbacks:**
   When running the validation loop (FR-054), if no specific configuration exists in `.bormagi/config.json` for validation commands, should we attempt to auto-detect common commands (e.g., if `package.json` has `lint`, run `npm run lint`), or should we remain passive and only run validation if explicitly configured?

3. **GitHub Authentication:**
   For `GitHubService`, should we rely on the host system's `gh` CLI for all operations (as is currently done in `git-server.ts`), or should we migrate to using the native VS Code GitHub Authentication provider (`vscode.authentication.getSession('github')`) and Octokit REST API for better reliability and lower dependency overhead?
